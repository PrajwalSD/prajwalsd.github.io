Good monitor for a MacBook Pro

November 20, 2021

I spend most of my time working on a code editor or a browser on my MacBook Pro (2017 model). Essentially all I consume at work on a computer screen is just characters mostly.

Before the COVID-19 pandemic, I never felt a need for a dedicated work setup at home, but this one didn’t seem like it was here to go anytime soon from our lives. So, I started looking for a perfect companion monitor for my laptop. To me, the definition of the word perfect here was apparent — it should be able to render/display text very well, like really well.

The obvious choice was to go for one of the Apple approved 4K/5K Ultrafine family of monitors made by LG and not worry about anything else due to Apple’s Retina display standards. Still, my wallet didn’t like the idea of it. So, my hunt for this perfect monitor continued, and after spending a week on this, I came across 31.5'' UltraFine™ UHD 4K Ergo IPS Monitor with USB Type-C™ monitor.

Although LG markets this monitor in the same family as the rest of the Ultrafine series monitors, the pixel density on this one is ~140ppi which falls short of Apple’s standards for a Retina display on a computer screen (i.e. >200ppi).

The build quality, USB-C, HDR10, 4K, ergonomic features, colour profiles, and the price point made the buy a bit more appealing than others in my little research. However, I was still a bit worried about the main thing I was looking for in a monitor, i.e. a monitor that can render text very well, which ultimately came down to the display panel’s pixel density for me.

I felt like I should give this monitor a try, so I bought one.

At first, I thought this was a mistake and wanted to return it right away — the text didn’t look crisp the way I wanted it. But, truth be told, everything else about this monitor was excellent and delivered on the promise. So somehow, I wanted to make this work.

macOS has fractional scaling, which means your text is going to look a little blurry in most of the default resolutions available for the display, using 1x scaling on a 32-inch 4K monitor was not an option for me.

So, I started playing with different display resolutions to try and see what mimics a “good” rendering for this monitor given the distance from the eye & the viewing angle of my setup. After much trial & error, I felt2304x1296 resolution worked very well for my setup (i.e., at 1.6666... scaling factor). Oh, I used QuickRes to try unlisted resolutions quite easily, by the way but you can even set it directly via System Preferences if you can’t be bothered to install an app.

I did try the different brightness/contrast/sharpness levels in my honest effort to find my perfect setting, but the default values always did the trick for me in most cases. I think the “Mode 4” Gamma mode worked best to compensate for the overly bright whites.

I also explored various levels of Font Smoothing using a simple app called Font Smoothing Adjuster, and to me default ‘medium’ setting looked the best.

Below is what I settled on to summarize it all:

  • Resolution set at2304x1296
  • Gamma mode set on “Mode 4”
  • Sharpness set at default
  • No changes to font smoothing (left it at default)

I have been on this configuration for a while and occasionally rant about the blurry text on a few apps but I think this is it, perhaps I will some day find a perfect monitor that fits my budget but for now I’m pretty happy with it.
